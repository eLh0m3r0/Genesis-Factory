# Validate Project

Check that a project has all required files and they're well-formed.

## Process

Check each of these and report status:

### Required Files
- [ ] `VISION.md` exists and has sections: Project, Direction, Technical Priorities
- [ ] `CLAUDE.md` exists and has sections: Stack, Architecture, Critical Rules
- [ ] `BACKLOG.md` exists

### Recommended Files
- [ ] `RESEARCH.md` exists (generated by /discover)
- [ ] `AGENTS.md` exists (generated by /retro)
- [ ] `heartbeat_config.yaml` exists (for monitoring)

### VISION.md Quality
- [ ] `project_weight` is set (1-5)
- [ ] Has at least 3 sentences of direction
- [ ] Technical priorities are listed

### CLAUDE.md Quality
- [ ] Stack/technologies are listed
- [ ] Critical rules are defined
- [ ] Test instructions are present

### BACKLOG.md Quality
- [ ] Stories have required fields: status, priority, effort, acceptance_criteria
- [ ] Story IDs follow format: `[PREFIX-NNN]`
- [ ] Acceptance criteria are specific and testable
- [ ] No orphaned "in_progress" stories (started but never finished)

### Git & GitHub
- [ ] Git repo initialized
- [ ] Remote configured: `git remote -v`
- [ ] CI workflow exists: `.github/workflows/ci.yml`

## Output Format

```
🔍 Validation — {project-name}

Files:
  ✅ VISION.md
  ✅ CLAUDE.md
  ✅ BACKLOG.md
  ⚠️ RESEARCH.md missing (run /discover)
  ⚠️ AGENTS.md missing (created after first /retro)
  ❌ heartbeat_config.yaml missing

Quality:
  ✅ VISION.md: project_weight=3, direction clear
  ⚠️ CLAUDE.md: missing "Critical Rules" section
  ✅ BACKLOG.md: 5 stories, all have required fields

Git:
  ✅ Remote: github.com/user/project
  ❌ No CI workflow (.github/workflows/ci.yml)

{total_pass} passed, {total_warn} warnings, {total_fail} issues
```

If issues found, suggest fixes:
- "Run /discover to generate RESEARCH.md"
- "Add a Critical Rules section to CLAUDE.md"
- "Create .github/workflows/ci.yml for CI/CD"
